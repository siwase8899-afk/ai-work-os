Project 02

AI Wellness Coach 기획

식단 관리는 사진을 분석하는 기능을 넘어, 기본 신체 정보와 운동·수면 데이터를 함께 보고 매일의 컨디션과 다음 행동을 제안하는 개인 건강 운영 에이전트로 발전시킨다. 현재는 초안 웹앱을 붙여두고, 이후 사진 분석과 주간 리포트로 확장한다.

Step 1

프로필 입력

Step 2

식사 사진 기록

Step 3

운동·수면 연결

Step 4

다음 행동 제안

Step 5

주간 리포트

입력 데이터

나이, 키, 몸무게, 목표, 활동량, 알레르기, 식사 사진, 운동, 수면 시간을 기록한다.

AI 분석 역할

칼로리, 탄단지, 수면 부족, 운동 후 회복 필요성, 과식/부족 패턴을 함께 추정한다.

안전 원칙

의학적 진단이 아니라 건강 루틴 기록과 의사결정 보조로 운영하고, 질환 관련 판단은 전문가 상담으로 넘긴다.

Roadmap

개발 로드맵

Project 01

주식 대시보드

  • 보유 종목, 매입가, 현재가, 수익률을 안정적으로 관리한다.
  • 필요 과정: 데이터 구조 확정, 시세 API 연결, 뉴스 수집, 지표 계산, 자동 업데이트.
  • 연결 후보: Yahoo Finance, GitHub Actions, 뉴스 RSS/API, Google Sheets.
Project 02

AI Wellness Coach

  • 식단, 운동, 수면 기록을 연결해 개인 건강 루틴을 관리한다.
  • 필요 과정: 프로필 입력, 음식 사진 분석, 영양소 추정, 수면/운동 데이터 연결, 주간 리포트.
  • 연결 후보: Vision API, Health 앱/Google Fit, 음식 영양 DB, Claude Code 에이전트.
Project 03

지사 / 콘텐츠

  • 지사별 요청사항, 이슈, 콘텐츠 일정과 성과를 업무 대시보드로 정리한다.
  • 필요 과정: 지사 데이터 템플릿, 콘텐츠 데이터 템플릿, 리포트 생성 규칙, 승인 플로우.
  • 연결 후보: Google Sheets, Gmail, Calendar, Drive, Slack/Teams, Notion MCP.
Final System

통합 AI Work OS

  • 회사와 개인 업무를 분리하되, 매일의 실행은 한 화면에서 운영한다.
  • 필요 과정: 프로젝트 허브, 데이터 분리, 자동화 로그, 인간 승인, 배포 구조 정리.
  • 연결 후보: GitHub Pages, GitHub Actions, Claude Code, Google Workspace MCP, 외부 API.

운영 원칙: AI는 수집, 분류, 요약, 분석, 리포트 초안을 맡고 최종 실행과 민감한 결정은 사람이 승인한다.
